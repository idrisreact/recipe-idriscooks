# Murakamicity Design System Usage Guide

This guide provides practical instructions for implementing the Murakamicity design system in your React components and applications.

## Installation & Setup

### 1. Tailwind CSS Classes

The design system uses Tailwind CSS utilities. Ensure your `tailwind.config.js` includes the Murakamicity color palette:

```javascript
module.exports = {
  theme: {
    extend: {
      colors: {
        primary: '#F20094',
        'primary-foreground': '#FFFFFF',
        accent: '#FF6B9D',
        muted: '#6B7280',
        'muted-foreground': '#9CA3AF',
        background: '#FAFAFA',
        surface: '#FFFFFF',
        border: '#E5E7EB',
      }
    }
  }
}
```

### 2. CSS Custom Properties

Add these CSS variables to your global styles:

```css
:root {
  --primary: #F20094;
  --primary-foreground: #FFFFFF;
  --accent: #FF6B9D;
  --muted: #6B7280;
  --muted-foreground: #9CA3AF;
  --background: #FAFAFA;
  --surface: #FFFFFF;
  --border: #E5E7EB;
}
```

## Animation Guidelines

### Hover Animations

Use subtle scale and shadow changes for interactive elements:

```css
.interactive-element {
  transition: all 200ms ease-out;
}

.interactive-element:hover {
  transform: scale(1.02);
  box-shadow: 0 10px 15px -3px rgba(0, 0, 0, 0.1);
}

.interactive-element:active {
  transform: scale(0.98);
}
```

### Loading States

Implement consistent loading animations:

```tsx
const LoadingSpinner = () => (
  <div className="w-5 h-5 border-2 border-current border-t-transparent rounded-full animate-spin" />
);

const LoadingButton = ({ loading, children, ...props }) => (
  <button className="murakamicity-button" disabled={loading} {...props}>
    {loading && <LoadingSpinner />}
    {children}
  </button>
);
```

## Accessibility Best Practices

### Focus States

Ensure all interactive elements have visible focus states:

```css
.custom-input {
  @apply border border-border rounded-lg px-3 py-2
         focus:outline-none focus:ring-2 focus:ring-primary 
         focus:border-transparent;
}
```

### Color Contrast

Always test color combinations for sufficient contrast:

```tsx
// Good contrast
<div className="bg-primary text-white p-4">
  High contrast text
</div>

// Avoid low contrast
<div className="bg-accent text-white p-4">
  ‚ùå Lower contrast - harder to read
</div>
```

### Semantic HTML

Use proper HTML elements and ARIA attributes:

```tsx
const AccessibleButton = ({ loading, children, ...props }) => (
  <button 
    className="murakamicity-button"
    aria-label={loading ? "Loading..." : undefined}
    disabled={loading}
    {...props}
  >
    {loading && <span aria-hidden="true"><LoadingSpinner /></span>}
    {children}
  </button>
);
```

## Testing Your Implementation

### Visual Testing

- Ensure consistent spacing using the 4px grid system
- Verify color contrast meets WCAG guidelines
- Test hover and focus states on interactive elements
- Check responsive behavior across different screen sizes

### Accessibility Testing

- Use screen readers to test navigation
- Verify keyboard navigation works properly
- Test with high contrast mode enabled
- Ensure all interactive elements have focus indicators
